**bench:** Composite-score rubric for bench results ([#1398](https://github.com/kwad77/pincher/issues/1398)). `cmd/pinch/bench_composite.go` ships `ComposeScore(inputs, weights)` and `ComposeBreakdown(inputs, weights)` — collapses multi-axis bench results (accuracy, tokens, latency p50/p95, round-trips-saved) into one comparable number per scenario so reviewers can say "composite X beats atomic chain Y by N points" without eyeballing three columns at once. Pattern ported from Stoa's autoresearch-score: weighted-sum where some axes are raw counts (more = better) and others are improvement-over-baseline gains (capped at zero so a regression contributes nothing rather than pulling the composite negative). Default weights pinned in `DefaultCompositeWeights` and asserted by `TestComposeScore_DefaultWeightsRanking` so weight tuning shows up in PR diffs. The rubric scaffolds the Phase 4 composite-tool work ([#1391](https://github.com/kwad77/pincher/issues/1391)) where each composite ships its own bench scenario.
