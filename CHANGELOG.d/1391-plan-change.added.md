**`plan_change` MCP tool — Phase 4 composite #2 ([#1391](https://github.com/kwad77/pincher/issues/1391) v0.82):** new pre-edit blast-radius composite. Takes a `target` (file path, symbol id, or free-form name), resolves it three ways (`::`-shaped id → `GetSymbolScoped`; recognized extension or path-separator → `GetSymbolsForFile`; otherwise BM25 search top callable hit), traces inbound `CALLS` callers up to depth 2 by default (clamped to 4), partitions them by depth (depth-1 = CRITICAL, depth-2+ = HIGH), flags each row with `is_test` (via the canonical `isTestFile` helper covering `__tests__/`, `tests/`, `_test.go`, `_spec.rb`, etc.) and `cross_package` (path-based: target dir ≠ caller dir), and looks up ADRs whose key or value mentions the target's package, directory components, or any affected symbol's name. Returns `{target, blast_radius, related_adrs}` with `blast_radius.summary` totals, an explicit `cross_package` list, and a sorted `test_files_intersecting` list so the agent knows what to run before pushing. Emits a `warnings_v2[code=blast_radius_high]` entry when depth-1 callers exceed 14 (per the roadmap example) with a "consider staged refactor" suggestion. Replaces the canonical pre-edit confirmation sequence: `changes(unstaged=true)` → `trace(seed, dir=in)` × N → `context(seed=top_caller)` → `adr(action=list)`. Read-only, no internal MCP round-trips per the composite contract. 14-test audit suite (positive + negative + control + cross-check) covers the path-vs-name heuristic, package-of-file extraction, missing target rich-error, target-not-found empty_reason, all three resolution paths, depth partitioning + cross-package flag, test-file intersection, ADR keyword overlap (positive and negative), high-blast-radius warning firing + sparse-target no-firing, and the depth=99→4 clamp. Sequencing: this is composite #2 of 5 in the Phase 4 roadmap (`docs/integrations/composite-tool-roadmap.md`); `audit_unused`, `onboard_module`, `why_empty` roll forward to v0.83-v0.85.
