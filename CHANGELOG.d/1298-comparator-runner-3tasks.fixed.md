Fixed and expanded the raw-Read/Grep comparator runner (`scripts/comparator-raw-readgrep.sh`) — #1298 §2 v0.89. The FILE-B runner (#1521, v0.86) never executed a single task: four bugs each aborted it before any measurement — (1) the gateway was started as `pin_resp=$(pincher … &)`, backgrounding inside the command-substitution subshell so `$!` was unbound and `set -u` killed the script; (2) `pincher --http` also runs the MCP stdio loop, and a backgrounded process whose stdin EOFs at once triggered the graceful stdio shutdown, exiting the whole process — needs `--no-stdio`; (3) `--http :0` binds all interfaces, which pincher default-denies without `--http-key` (#199) — needs `--http 127.0.0.1:0`; (4) `pincher --data-dir D index PATH` put `--data-dir` in argv[1] so the `index` subcommand was never detected and pincher ran as the MCP server, indexing nothing — the subcommand must lead. The `continue-on-error: true` advisory `External comparator` workflow masked the breakage every Saturday since v0.86. The runner now actually works and grows the canonical-workflow corpus from the 1 placeholder task to 3 agent-loop shapes — find-symbol-by-name (search vs `grep` output), read-with-context (context vs whole-file `cat`), find-callers (pinchQL `query` vs `grep` call sites + per-file reads) — each recording pincher-vs-raw wall-clock and a byte token-proxy. Output JSON bumps to `schema_version: 2`.
