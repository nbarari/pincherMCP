Branch-drift advisory now also skips projects whose path points to a file rather than a directory (#1669 v0.87). Same-release dogfood-found cascade: after #1667 added the path-existence check, the dogfood machine's surviving "drift" was a `probe` project whose original directory had been overwritten by a same-named 3.6 MB executable. `os.Stat` saw a valid file (not IsNotExist), so #1667's IsNotExist check passed through; `git -C <path-to-file>` then resolved via the PARENT repo's .git and reported the parent's current branch as the project's branch — false-positive drift. Fix extends the check to `err != nil || !fi.IsDir()` so any non-directory path skips silently. Same routing semantic: those are dead-path ghosts for `list prune_dead=true`, not the index-refresh remediation. Both MCP and CLI doctor copies updated.
