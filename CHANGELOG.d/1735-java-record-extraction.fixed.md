Java `record` declarations are now extracted as types, not methods (#1735 v0.89). The Java `classRE` matched only `class`, so a `record Point(int x, int y)` line fell through to `funcRE` — which read `record` as a return-type token and emitted a phantom `Method` named `Point`. The record surfaced under `kind=Method` (wrong for `search`/`trace`/`dead_code`/`audit_unused`) and never under `kind=Class`, and its own methods lost their `Method` classification for want of a parent type. `classRE` now matches `(?:class|record)`; claiming the line suppresses the funcRE match (the same mechanism C# records already rely on). Found dogfooding the extractor across languages the #1389 cross-language sweep did not cover.
