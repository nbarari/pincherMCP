**Plugin-extractor surface deferred to v1.x (FILE-V, #1540).** ADR-0003 ([docs/adr/0003-plugin-extractor-deferral.md](docs/adr/0003-plugin-extractor-deferral.md)) records the v0.88 design-review decision: defer #1333 (custom language extractor plugin point) to v1.1.0 rather than freezing the API at v1.0. Rationale: extractor interface still hardening across v0.67 / v0.73 / v0.76 (`scopeRE`, universal regex-tier promotion, `USES_VAR`), `buildmode=plugin` fragility (Linux/macOS only, exact host-version match, no cross-compile), no external demand signal, and v1.0 critical-path resource concentration. Pincher 1.0 ships with a smaller, more honestly-frozen public surface; plugin API stays `experimental` per ADR-0002.
