Memory-pressure-aware watcher backoff (#1572 v0.87, Linux first slice). When a host is under memory pressure, the file-watcher's 5-second tick previously kept poking the indexer on the same cadence — a large project on a small machine could OOM-kill mid-extraction with no signal beyond "the process died." The watcher now reads available system memory before each tick and skips the tick (logging `pincher.watcher.memory_backoff` at `warn`) when it's below a configurable floor. The floor is `PINCHER_WATCHER_MEMORY_BACKOFF_MIB` (default 512 MiB). This first slice implements the Linux reader only — `/proc/meminfo`'s `MemAvailable`, pure-file, no syscalls or cgo. On macOS and Windows the reader reports unavailable and the watcher never backs off, preserving prior behaviour (a safe no-op rather than a wrong guess); their platform-specific readers (`vm_stat` / `GlobalMemoryStatusEx`) are follow-up slices. Documented in REFERENCE.md alongside the other resource-pressure env vars.
