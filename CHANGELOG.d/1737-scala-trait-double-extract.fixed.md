Scala `trait` declarations are no longer double-extracted (#1737 v0.89). `scalaRE.classRE` listed `(?:class|object|trait)` while a separate `interfaceRE` also matched `trait`, so a plain `trait Store` line matched both — emitting one `Store#Class` AND one `Store#Interface` for a single declaration, a conflicting-kind duplicate (and `search` surfaced the trait under both `kind=Class` and `kind=Interface`). `trait` now belongs to `interfaceRE` only; `interfaceRE` gained the modifier prefix `classRE` already carried (`sealed`/`abstract`/`final`) so `sealed trait` still resolves rather than matching neither regex. `interfaceRE` sets the scope tracker, so trait-scoped methods still parent correctly. Found dogfooding the extractor across the languages the #1389 cross-language sweep did not cover.
