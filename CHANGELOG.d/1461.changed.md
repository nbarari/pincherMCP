PHP extractor promoted from confidence **0.70 → 0.85** (stable-regex tier). Adds the major declaration kinds modern PHP (8.x+) codebases use: `interface` and `trait` (PHP's code-reuse mechanism, modeled via `scopeRE` so trait methods scope as Method/Parent=Trait without emitting a separate Class symbol — the trait isn't a type, just a mixin container); `enum` (PHP 8.1+) covering both pure enums (`enum Status { case Active; }`) and backing-type enums (`enum Priority: int { case High = 3; }`); `#[Attribute]` and `#[Attribute(args)]` prefix tolerance (PHP 8+ uses `#[...]` syntax — covers `#[Route]`, `#[ORM\Entity]`, `#[Required]`, `#[Authorize(role: "admin")]`, `#[Test]`, etc.); `final` and `readonly` class modifiers (PHP 8.2+); `abstract` modifier on `funcRE` (was only on classRE pre-fix); **flexible modifier ordering** — PHP grammar allows access modifiers (`public`/`private`/`protected`) and class-level modifiers (`static`/`final`/`abstract`/`readonly`) in any combination via a single order-independent alternation, so `abstract public function` and `public abstract function` both extract correctly. Pre-fix modern Symfony / Laravel / Doctrine / PHPUnit codebases were heavily under-extracted: every interface invisible, every trait invisible, every PHP 8.1+ enum invisible, every `#[Attribute]`-annotated class/method dropping the annotated line entirely.
