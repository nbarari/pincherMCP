**External comparator bench (FILE-B, #1521).** `scripts/comparator-raw-readgrep.sh` runs pincher tool calls side-by-side with the equivalent "raw Read+Grep agent loop" (grep + cat) and emits JSON capturing wall-ms + bytes-read for each side per task. The bytes-ratio anchors the token-savings methodology (FILE-A, #1520) with measured numbers rather than the `tokens_saved` envelope's known-overcounting estimate. `.github/workflows/comparator.yml` runs Saturday 15:00 UTC + `workflow_dispatch`. v0.86 ships one representative task (`find-symbol-by-name`) as the working reference; tasks 2-N (read-with-context, find-callers, multi-symbol-aggregate) ship in v0.87-v0.90 follow-ups one per minor — each task is non-trivial to write defensively, all-or-nothing bundling would block on the slowest. Methodology + intentional scope-cut rationale in [docs/methodology/external-comparator.md](docs/methodology/external-comparator.md). Phase 1 (v0.86-v0.90): advisory. Phase 2 (v0.91+): hard floor on bytes_ratio per FILE-A acceptance.
