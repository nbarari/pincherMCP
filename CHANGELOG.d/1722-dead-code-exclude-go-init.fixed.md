`dead_code` and `audit_unused` no longer flag Go `init` functions as dead code (#1722 v0.89). Go `init` functions are invoked by the runtime at package initialization, never by an explicit call — they carry zero inbound CALLS edges by language design, so `GetDeadCode` reported every `func init()` as a high-confidence safe-to-delete candidate (and `audit_unused`, which builds on the same SQL path, classified them `confidence: high` / `deep_trace_confirms_unreachable`). Acting on that advice breaks package initialization — pincher's own `internal/ast/*.go` `init()` functions register the language extractors, so "deleting" them silently disables Bash/HCL/HTML/Jinja/Markdown extraction. `GetDeadCode` now carves out Go `init` (`s.language = 'Go' AND s.name = 'init'`), mirroring the existing interface-dispatch carve-out (#493) and the qualified_name_collision `init` carve-out (#1696). Found by dogfooding `audit_unused` against pincher-repo.
