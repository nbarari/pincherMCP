Fixed the `host-conformance` CI gate, which was red on master with all 6 hosts reporting "pincher produced no responses" (#1706 v0.89). `scripts/host-conformance.sh` piped each host's recorded JSON-RPC request stream into pincher via `< in_stream`, which EOFs stdin the instant the bytes are delivered. A real MCP host keeps the stdio session open for its lifetime and never pipes-then-immediately-closes; on an instant EOF the SDK's stdio loop can tear the session down before the in-flight request handlers flush their responses, so the replay captured zero output. The harness now feeds the requests and then holds stdin open for `HOSTCONF_STDIN_HOLD` seconds (default 3) before EOF — mirroring a live host and giving pincher its sub-millisecond answer window before the graceful EOF shutdown. Pure test-harness change; no product code touched. All 6 hosts now replay the full 4-request canonical workflow and the `tools/list` assertion passes. The separate scripted-`echo | pincher` EOF-drain robustness (a real but non-release-blocking fragility) stays tracked in #1706.
