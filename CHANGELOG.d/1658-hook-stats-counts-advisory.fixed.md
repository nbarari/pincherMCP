Hook-stats aggregates now count `redirect_advisory` decisions (#1658 v0.86). After #1654 / #1656 split blocking redirects into `redirect` (legacy) and `redirect_advisory` (new), four SQL queries still hardcoded `decision = 'redirect'`: `HookConversionRate7d`, `HookCountsByTool7d`, `HookOverrideRate7d`, and `ResolveHookInvocationsForSession`. Once existing legacy `redirect` rows aged out of the 7-day window, conversion-rate and per-tool counts would silently go to zero even though the hook was firing on every Read/Grep. Each query now uses `decision IN ('redirect','redirect_advisory')` so both decision kinds aggregate together. Telemetry continuity preserved across the #1654 transition.
