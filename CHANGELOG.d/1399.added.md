New `pincher verify` subcommand re-hashes every indexed file's on-disk content with xxh3 (the indexer's hash function) and compares against the stored `files.hash` column. Surfaces three drift classes: `drifted` (file modified out-of-band since last index — symbol byte-offsets may now point at wrong content), `missing` (stored hash present but file deleted on disk), `unreadable` (permission / I/O error). Exit code 2 when any class is non-empty so CI / hooks can branch. Foundation-layer hygiene tool — doesn't auto-fix anything; the operator re-indexes a drifted project to bring its symbol-store back in sync. Mirrors the Stoa-family `verify`/`doctor`/`probe` trinity pattern: pincher already had `doctor` (advisories); this issue adds the `verify` leg (integrity-check). Flags: `--json` for structured output, `--project NAME|ID|SUBSTRING` to restrict the sweep. New `Store.ListFilesWithHashesForProject()` reader-pool method batches the per-project file/hash readout in one SELECT — N round-trips would dominate latency on warp_rc / Codex-scale projects (4k+ files). Live dogfood at filing: pincher-repo + its testdata corpora showed 716 checked / 675 in sync / 18 drifted / 23 missing — exactly the kind of mid-session edit drift the tool is designed to catch. Closes #1399.
