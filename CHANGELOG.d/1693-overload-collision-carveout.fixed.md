`qualified_name_collision` doctor diagnostic no longer false-fires on legitimate method overloads (#1693 v0.89). Final fix from the #1389 cross-language sweep. pincher's qualified name is `Class.Method` with no arity/signature component, so every method-overload set in an overload-capable language (C#, Java, C++, Swift, Kotlin) collided on QN — `Calc.Add` ×3 for `Add(int,int)` / `Add(int,int,int)` / `Add(float,float)`. disambiguateDuplicates suffixes the duplicates `~<line>` so all symbols survive correctly, but the diagnostic still recorded an extraction failure, eroding trust in `pincher doctor` on every C#/Java/C++ codebase that uses overloading. recordExtractionHeuristics now skips a collision that is a legitimate overload set: every colliding symbol a Method/Function occupying a distinct, non-overlapping byte range. A true extractor duplication — overlapping or identical ranges, or a non-method sharing the QN — still flags. Mirrors the Go multiple-`init()` carve-out. Reindex impact: fools-gold-pirate 9→0, Yevolve 1→0 qualified_name_collision rows; the remaining sweep residual is now zero across all C#/Go corpora.
