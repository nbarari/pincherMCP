Schema migration language-scoped selective invalidation lands the v0.83 #1497 follow-up (#1543 v0.84). When a binary upgrade ran schema migrations classified as `MigrationInvalidates.Languages` (not `All`), the indexer's drift path now calls `Store.ClearFileHashesByLanguage(projectID, languages)` and runs a normal pass — only files of the listed languages re-extract; every other file keeps its hash and skips. Falls back to the full force-reindex if the targeted clear errors so under-invalidating stays impossible (silent correctness > perf miss).
