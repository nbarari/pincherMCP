`pincher doctor` now surfaces a `PreToolUse hook is NOT installed` advisory when Claude Code looks present (`.claude/` directory in cwd, `~/.claude/settings.json` exists, or `CLAUDE_CONFIG_DIR` env set) AND no `pincher hook-check` PreToolUse entry is detected in any known settings location (#1635 v0.85). Closes the loop with #1632: a user can now see both that pincher gets called heavily (`pincher stats`) AND that the runtime Read/Grep redirect (#627) is OFF (`pincher doctor`). Without the advisory the hook silently never gets installed and `hook_invocations` stays empty even on installs with thousands of MCP calls. Remediation: `pincher init --target=claude`. Mirrors silent-confidently-wrong design class — surfaces the gap with a concrete action rather than hoping users read CLAUDE.md.
