**Per-language v1.0 fitness declaration ([#1533 FILE-N](https://github.com/kwad77/pincher/issues/1533) v0.86):** REFERENCE.md gains a "v1.0 fitness declaration" subsection mapping every supported language to one of four explicit statuses — `promised` (10 languages: Go, Python, YAML/JSON, Bash, HCL, TOML, Markdown, Jinja2, JS/JSX); `supported` (10 languages: TS/TSX, Rust, Java, C#, PHP, Kotlin, Swift, C, C++, Makefile, SQL — stable-regex tier); `best-effort` (7 languages: Ruby, Scala, Lua, Zig, Elixir, Dart, R — approximate-regex tier); `excluded` (Haskell, stub). Each row carries the known limitation so a v1.0 user asking "is Ruby supported?" gets the honest answer: yes, with caveats. Per ADR-0002 the *tier* assignment and *v1.0 status* are part of the frozen v1.0 surface — promotion (Rust regex 0.85 → AST 1.0) is a minor additive feature in 1.x; demotion (a language going DOWN) is a 2.0 breaking change. Pre-fix, the existing 9-axis capability matrix surfaced WHAT works per language but never declared WHICH bar pincher will hold to release. Closes a v1.0-credibility gap: claims like "we support Ruby" can now point at this table for the exact answer.
