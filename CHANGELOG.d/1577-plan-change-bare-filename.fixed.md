**plan_change: bare-filename target falls through to BM25 instead of silent-empty (#1577).** A target like `charge.go` matched `looksLikeFilePath` (extension), so the handler routed to the file_path branch — but `GetSymbolsForFile` against the bare basename never matches the indexed full-path `payments/charge.go`. The handler then hit the empty-suspects path with diagnosis "target did not resolve" — silent-confidently-wrong, because BM25 search would have found the right symbols. Fix: when the file_path branch returns zero hits AND the target has no slash, strip the extension and re-issue as a BM25 search. Real root-level files like `main.go` still resolve via the file branch (their indexed path IS the bare name). Regression test pins the fallback. Part of composite-hardening umbrella #1582.
