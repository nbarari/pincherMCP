TS/JS regex extractor handles nested functions correctly. Pre-fix `#1375`'s single-slot most-recent-function tracker got overwritten by inner `async function` declarations; after the inner ended, all of the outer's remaining Variables fell back to module scope, and sibling inner functions' Variables collided at the module level (every Next.js App Router `page.tsx` with two `fetch` helpers lost symbols silently to the `qualified_name_collision` dedup guard). Replaced with a stack: push on every funcRE match, pop entries whose endLine the current line has passed, scope nested functions + Variables to `stack.top.qn` (innermost active function). Sibling inner functions get distinct chained QNs (`page.Page.fetchRuns.res` vs `page.Page.fetchWorkflows.res`) (#1422).
