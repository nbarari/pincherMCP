**onboard_module: rows.Err() now hard-errors instead of silently dropping partial scope (#1581).** Previously `handleOnboardModule` had `if err := rows.Err(); err != nil { /* defer to empty-handling branch */ }` — but the empty branch only fires when `len(scopeSyms) == 0`. If scanning produced some symbols and then errored, the composite continued with partial data, producing a misleading language breakdown, missing entry-points, undercount in `ExportedSurfaceCount`, and stale next-step IDs. Now returns `errResultRich` so the agent knows to retry. Same silent-confidently-wrong family that was supposedly drained in v0.59. Part of composite-hardening umbrella #1582.
