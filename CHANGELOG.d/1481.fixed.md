**fix(ast/markdown):** Intra-doc REFERENCES edges use the H1-heading slug as the QN root (not the filename) so they actually match Section symbols ([#1481](https://github.com/kwad77/pincher/issues/1481)). Pre-fix the link walker passed `result.Module` (= filename basename like `"README"`) as `selfModule` to `canonicalDocsLinkTarget`, so `[Configuration](#configuration)` emitted ToName=`"README.configuration"`. But Section symbols are built from the H1-heading hierarchy (`# Project` → QN root `"project"`), so the actual Section's QN was `"project.configuration"`. Shape mismatch → 100% of intra-doc REFERENCES edges dropped at the resolver. Unit test `TestExtractMarkdown_IntraDocLink_EmitsREFERENCES_1343` was buggy too — it asserted the filename-rooted shape that doesn't match any Section, but it still passed because it never round-tripped through the resolver. Post-fix `selfModule` derives from `headings[0]` (the first heading's slug); the test renamed to `_1343_1481` and updated to assert the H1-rooted shape that actually resolves. Pinned by `TestIndex_MarkdownIntraDocREFERENCES_PersistedToDB_1481` (integration probe — 3-section README with 3 intra-doc links produces exactly 3 REFERENCES edges in the DB; 0 pre-fix).
