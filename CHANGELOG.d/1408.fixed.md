`symbol` and `context` now scope to the session project FIRST when no `project=` arg is set, falling back to unscoped only when the id isn't in the session project. Pre-fix the unscoped lookup could return a fork's row when the same id lived in both the session project and a fork (e.g. a `d:\codex\sniffer`-shaped mirror of `pincher-repo`), and the strict-cross-project guard (#1232) then erroneously rejected the call with "exists only in project X" — even though the symbol DID also live in the session project. This broke the canonical `search → symbol` workflow whenever any indexed fork shared identifier shapes with the session project. (#1408)
