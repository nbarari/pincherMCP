**Time-to-first-success bench scaffolding (FILE-Q, #1535).** `scripts/time-to-first-success.sh` measures the brand-new-user path (`git clone` → `pincher index` → `pincher search`) in four timed phases, reporting JSON with phase-level + total milliseconds and a budget pass/fail boolean. The `.github/workflows/time-to-first-success.yml` workflow runs weekly (Monday 10:00 UTC) and on `workflow_dispatch`; it fails when `total_ms` regresses >20% over the committed baseline at `testdata/bench/time-to-first-success.bench.txt`. Methodology + baseline-refresh procedure documented in [docs/methodology/time-to-first-success.md](docs/methodology/time-to-first-success.md). Per #1535, the published budget is **≤5 minutes on a ~5k-file repo**. Pairs with FILE-R (#1536) host-conformance matrix that will use the same script per supported MCP host.
