**Environment-variable doc parity ([#672](https://github.com/kwad77/pincher/issues/672) workstream 4 v0.79 hardening):** new `TestEnvVars_TutorialMentionsDocumentedInReference` scans `docs/tutorials/*.md` + `docs/deployment/*.md` for `PINCHER_*` env-var mentions and asserts each appears in `docs/REFERENCE.md`. Caught four drifts: `PINCHER_DATA_DIR` (recommended in vscode-copilot tutorial) was never documented; `PINCHER_HTTP_ALLOW_OPEN` + `PINCHER_MCP_HTTP_PATH` (used in http-dashboard / codex / docker / homebrew docs) were also undocumented in REFERENCE.md's env-var table; and the user-facing shell variable `$PINCHER_KEY` in the http-dashboard tutorial misleadingly looked like a pincher env var — renamed to `$KEY` to make its user-scope intent obvious. Doc rows added for all three real env vars; the test prevents the same drift recurring.
