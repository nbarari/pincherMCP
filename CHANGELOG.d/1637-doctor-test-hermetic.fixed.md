`go test ./cmd/pinch/` is now hermetic against a developer's real `~/.claude/` directory. After #1635/#1636 landed the PreToolUse-hook advisory, `TestDoctorReport_EmptyDatabase` + `TestDoctorReport_BinaryVersionPopulated` failed on any contributor whose home directory had `~/.claude/settings.json` present, because the advisory's Claude-Code detection fired in the test process. A `withNoClaudeCodeDetected(t)` helper now neutralizes HOME / USERPROFILE / CLAUDE_CONFIG_DIR for those tests. CI was green because GitHub Actions runners have no `~/.claude/` — the gap was only visible to humans running tests locally (#1637).
