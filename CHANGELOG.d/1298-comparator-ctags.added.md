Added universal-ctags as a second comparator in the code-intelligence comparator harness (#1298 §2 v0.89), completing the "Read/Grep baseline + at least one external tool" acceptance. `scripts/comparator-raw-readgrep.sh` now measures each canonical task against pincher, the raw Read+Grep loop, AND a ctags symbol-definition index, emitting `ctags_ms` / `ctags_bytes` alongside the existing columns (`schema_version: 3`; null when ctags is absent or structurally cannot answer the task). ctags is install-light, offline, and deterministic — a fair stand-in for "a real code-intel tool that is a tags index." The asymmetry is the point: ctags answers find-symbol compactly (a tag record, comparable to pincher's layer 1), but has no call graph so it cannot answer find-callers at all, and extracts no symbol bodies so read-with-context still costs a whole-file read — isolating exactly what pincher's layer 2 (knowledge graph) and layer 3 (byte-offset retrieval) add over a plain tags index. `.github/workflows/comparator.yml` installs `universal-ctags` alongside `jq`.
