**v1.0 frozen surface declaration ADR + semver promise ([#1530 FILE-K](https://github.com/kwad77/pincher/issues/1530) + [#1531 FILE-L](https://github.com/kwad77/pincher/issues/1531) v0.84):** new `docs/adr/0002-v1-frozen-surface.md` enumerates every surface element pincher's v1.0 promises and assigns it a status (`frozen` / `evolving` / `experimental`). Frozen: 26 MCP tool names; tool input/output JSON Schemas; `_meta` envelope with `_v2` extension-point pattern; HTTP gateway routes (`/v1/openapi.json`, `/v1/<tool>`, `/v1/hook-stats`, `/v1/architecture/<aspect>`, `/v1/tool-payload-stats`); 15 CLI subcommands + their flags; symbol ID format. Evolving: SQLite schema (forward-only migrations with `invalidates` classification); pinchQL grammar. Experimental: MCP `resources` capability if it ships (#1083); plugin extractor API if it ships (#1333 / FILE-V). The freeze checkpoint is v0.84.0 — after that, every PR touching a frozen surface element either reverts or routes to 2.x. CONTRIBUTING.md gains a corresponding "Semver in pincher 1.x" section enumerating: what is a breaking change (renaming a tool, removing a `_meta` field, changing an enum value, etc.); what is a minor (additive — new tool, new field, new language extractor, schema migration); what is a patch (bug fix that doesn't change the published surface); the deprecation-cycle rule (one minor of `Deprecated:` warnings before flag / field removal); the PR-template checkbox that catches accidental frozen-surface breakage at review. CI gates (`TestToolContract_GoldenFile`, `TestRegisterTools_Annotations_EveryToolHasOne`, `TestMCPSurface_AllRegisteredToolsAgentCallable`, `TestReferenceMD_EveryCLISubcommandHasSection`) are now load-bearing for v1.0 compatibility — accidental drift breaks CI before it ships.
