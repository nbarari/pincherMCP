**Observability deployment guide ([#1526 FILE-G](https://github.com/kwad77/pincher/issues/1526) v0.87):** new `docs/deployment/observability.md` unifies the three production observability surfaces that landed piecemeal: Prometheus metrics at `GET /v1/metrics` (always on with `--http`, scrape-compatible with any Prometheus-flavoured collector, honors `--http-key`); OTLP traces (opt-in via `OTEL_EXPORTER_OTLP_ENDPOINT` per the standard OTel SDK env-var contract, including `OTEL_EXPORTER_OTLP_HEADERS` for auth and `OTEL_TRACES_SAMPLER` for sampling); structured slog logs (`PINCHER_LOG_FORMAT=json`, `PINCHER_LOG_LEVEL=debug|info|warn|error`, standard slog JSON contract). Document declares the 7-metric inventory as part of the v1.0 frozen surface per ADR-0002 — `pincher_tool_calls_total`, `pincher_tool_latency_seconds`, `pincher_tool_tokens_saved_total`, `pincher_index_files_total`, `pincher_index_symbols_total`, `pincher_db_size_bytes`, `pincher_wal_size_bytes`. Includes a Prometheus scrape-config example, the trace-span hierarchy (root `pincher.tool.<name>` → child `db.query.<op>` with `slow=true` when above the slow-query threshold), and the "is observability working?" quick-check commands. What's NOT in v1.0 observability scope (auto-discovery via SRV, tail sampling at the pincher layer, distributed-trace propagation across `fetch` → external service, dashboard trace-ID click-through) is enumerated separately so an operator's expectation is calibrated up-front.
