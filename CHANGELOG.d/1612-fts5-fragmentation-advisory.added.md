`pincher doctor` surfaces an FTS5 fragmentation advisory when any per-corpus vtab's data/idx-row ratio exceeds 10x (#1612 v0.87). The per-corpus FTS5 indexes accumulate micro-segments as INSERTs land — BM25 search latency degrades long before the user notices a slow `search`. The data/idx-row ratio is the cheapest accurate fragmentation indicator: post-rebuild corpora sit at 1–3x, and observed live fragmentation on a long-running install hit 62.8x on the config corpus. New `db.Store.FTS5Fragmentation()` returns per-corpus stats from the FTS5 shadow tables; `fts5FragmentationAdvisory` formats the user-facing message with the specific corpus name, current ratio, and `rebuild_fts` remediation. Advisory is mirrored to both the MCP `doctor` tool and the CLI `pincher doctor` per the CLAUDE.md bounded-duplication convention. Best-effort: a missing shadow table (pre-v9 schema or partial migration) returns no rows and the advisory stays silent. Closes the v0.85 piece-2 deferral of #1612 (piece 1 — bench coverage — shipped in v0.85).
