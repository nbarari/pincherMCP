`pincher index` CLI no longer silently discards slog output (#1621 v0.85). Pre-fix `runIndexCLI` called `log.SetOutput(io.Discard)` to silence stdlib log noise, but in Go 1.25 the default `slog.Default()` handler routes through `log.Default()`'s writer — so the redirect ALSO silenced slog. Net effect: v0.85's #1613 observability (`pincher.index.extraction.summary`, `pincher.resolve_block.summary`, `pincher.{imports,calls,reads,uses_var}.resolve.summary`, `pincher.index.gc.summary`) was invisible at the CLI surface. Removed the log redirect; slog now reaches stderr per Go's default. Scripts that want the old quiet behavior can `2>/dev/null`. `TestIndexCLI_Binary_JSONSummary` updated to use stdout-only capture instead of CombinedOutput — `--json-summary`'s stdout contract is single-JSON, and the new stderr noise is part of the diagnostic surface, not the output contract.
